# 要件定義書：チーム日報管理システム

## サービス概要

チームで使用する日報管理Webアプリケーションを開発する。

本システムは、メンバーが日々の業務内容を日報として記録し、チーム内で共有できるサービスである。また、投稿された日報に対してコメントを行うことで、迅速なフィードバックを可能にする。

紙やExcelで管理されがちな日報をWebアプリで一元管理し、業務の可視化とチームコミュニケーションの活性化を目的とする。

使用技術：

- Next.js（App Router）
- TypeScript
- Supabase
- Tailwind CSS
- Vercel

---

## ターゲットユーザー

### 主な対象ユーザー

- 中小企業のチームリーダー・マネージャー
- リモートワークを行うチームメンバー

### 利用シーン

- 日々の業務報告の記録
- チームメンバーの業務状況の把握
- 日報へのフィードバック共有

---

## 解決する課題

本システムは以下の課題を解決する。

### 1. 日報管理の非効率性

紙やExcelで日報管理している場合、検索や管理が困難になる問題がある。

→ Webアプリで一元管理することで解決する。

### 2. 業務の可視性不足

チームメンバーの作業内容が見えにくい問題がある。

→ 日報を共有することで業務内容を可視化する。

### 3. フィードバックの遅れ

日報へのコメントが遅れる問題がある。

→ コメント機能により迅速なコミュニケーションを可能にする。

## 必須機能一覧

### 1. ユーザー認証機能

- 氏名とメールアドレスとパスワードによる新規登録
- ログイン機能
- ログアウト機能
- パスワードリセット機能
- プロフィール編集（名前・アバター画像）
- 未ログイン時のログイン画面へのリダイレクト

### 2. チーム（部屋）機能

- 部屋作成
  - 作成フロー：
  
  ・「チーム参加」画面から「チーム作成」ボタン押下
  
  ・チーム名を入力し、作成ボタン押下

  ・teamsへ登録

  ・team_code発行

  ・team_membersへ登録
  
  ・チーム画面へ遷移

- 部屋参加（参加コード）
  - 参加フロー：

  ・「チーム参加」ボタン押下

  ・参加コード入力フォーム表示

  ・team_code一致確認
  
  ・team_membersへ登録

- 部屋切り替え
  - 自分が入っている部屋の一覧をセレクトボックスで選択し、切り替え。
- 同一部屋のメンバー一覧表示

### 3. 日報管理機能（CRUD）

#### 日報作成機能

- タイトル入力（必須・50文字以内）
- 日付入力（必須・デフォルトは当日）
- カテゴリ選択（開発・会議・その他等）
- 本文入力（必須・2000文字以内）
- 公開範囲の選択（チーム内 / 全体）
- バリデーションエラー表示

#### 日報一覧表示機能

- 選択中の部屋の日報表示（チーム内公開）
- 全体公開の日報も表示（部屋を跨いで閲覧可能）
- チームを選択した場合、チーム別に表示
- 作成日時の降順表示
- 表示項目：
  - タイトル
  - 投稿者名
  - 日付
  - カテゴリ
  - 本文冒頭
  - コメント数
  - 作成日時
- ページネーションまたは無限スクロール

#### 日報詳細表示機能

- タイトル
- 投稿者名
- アバター
- 日付
- カテゴリ
- 本文全文
- 作成日時
- 更新日時
- コメント一覧
- 自分の日報のみ編集・削除ボタン表示

#### 日報編集機能

- 自分が作成した日報のみ編集可能
- 公開範囲も編集可能（チーム内 / 全体）

#### 日報削除機能

- 自分が作成した日報のみ削除可能
- 削除確認ダイアログ表示

### 4. 検索・フィルター機能

- 日付範囲検索
- ユーザー絞り込み
- カテゴリ絞り込み
- キーワード検索（タイトル・本文・投稿者名の部分一致）
- 公開範囲フィルター（チーム内 / 全体）

### 5. コメント機能

- 日報へのコメント投稿（500文字以内）
- コメント一覧表示
- コメント削除（自分の投稿のみ）

### 6. エラーハンドリング・バリデーション機能

- APIエラー処理
- エラーメッセージ表示
- フロントエンドバリデーション
- Supabase RLSによる権限制御

## 非機能要件

### 認証・セキュリティ

- Supabase Authによる認証管理
- Row Level Security（RLS）によるアクセス制御
- 未ログイン時のアクセス制御

### レスポンシブ対応

以下の画面サイズに対応する：

- スマートフォン（375px以上）
- タブレット（768px以上）
- デスクトップ（1024px以上）

### パフォーマンス

- ページネーションまたは無限スクロールによる負荷軽減
- 適切なデータ取得処理の実装

### 可用性

- エラー発生時もアプリが停止しない設計
- 404ページの実装

### 開発品質

- TypeScript strictモード使用
- any型の使用禁止
- テストコード実装（Vitest・Playwright）
- CI/CD（GitHub Actions）

---

## 実装しない機能（スコープ外）

以下は必須機能には含めない：

- 日報へのいいね機能
- 日報テンプレート機能
- Markdown対応
- 画像アップロード機能
- 通知機能（メール通知等）
- 日報統計ダッシュボード
- CSVエクスポート機能
- ダークモード
- 多言語対応

---

## 追加した機能（必須ではないが必要と判断）

要件定義当初は「全体公開（日報の共有）」を前提としていたが、運用上「同一部屋内のみ共有」も選べると便利だと判断したため追加する。

- 日報作成時に、公開範囲（チーム内 / 全体）をスライドボタン等で切り替えられる
